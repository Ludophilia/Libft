Libft - ft_memchr 游릭游댮

Evil tests made with 游땓游땓游땓 passion

Context : The memchr() function scans the initial n bytes of the memory
       area pointed to by s for the first instance of c.  Both c and the
       bytes of the memory area pointed to by s are interpreted as
       unsigned char.

Test material
	- arr of type char or int ex : "" , { 0 1 2 3 4 }
	- Original memchr (#include <string.h>) for comparison

Prototype
	void	*ft_memchr(const void *s, int c, size_t n)

[游릭] Happy path
	ft_memchr(arr, 3, 5 * sizeof(int))
		-> (address of 3)
	ft_memchr("hello", 'l', 5 * sizeof(char))
		-> "llo"

[游릭] Empty str or arr
	ft_memchr("", 'l', 1 * sizeof(char))
		-> Null
	ft_memchr(arr, 7, 1 * sizeof(int))
		-> Null

[游릭] Null address
	ft_memchr((char *)0, 'W', 1 * sizeof(char))
		-> Segfault ?
	ft_memchr((int *)0, 555, 20 * sizeof(int))
		-> Segfault

[游릭] Modulo what??
	ft_memchr("hello", 'l' + 16 * 256, 5 * sizeof(char))
		-> "llo"
	ft_memchr(arr, 3 + 256, 5 * sizeof(int))
		-> (address of 3)

[游릭] Nowhere to be found
	ft_memchr("hello", 'x', 5 * sizeof(char))
		-> Null
	ft_memchr("hello", -2147483647, 3 * sizeof(char))
		-> Null
	ft_memchr(arr, -1, 5 * sizeof(int))
		-> Null
	ft_memchr(arr, 4242, 3 * sizeof(int))
		-> Null

[游릭] No size
	ft_memchr("hello", 'l', 0 * sizeof(char))
		-> Null
	ft_memchr(arr, 0, 0 * sizeof(int))
		-> Null

[游릭] Too many size
	ft_memchr("hello", 'l', 10 * sizeof(char))
		-> "llo"
	ft_memchr("hello", 'x', 10 * sizeof(char))
		-> Null
	ft_memchr(arr, 7, 10 * sizeof(char))
		-> Null
