Libft - ft_calloc 游릭游댮

Evil tests made with 游땓游땓游땓 passion

Context : The calloc() function allocates memory for an array of nmemb
       elements of size bytes each and returns a pointer to the
       allocated memory.  The memory is set to zero.

Test material
	- Array 1 of char of 5 elements or space allocated via malloc
	- Original calloc (#include <string.h>) for comparison
	- SIZE_MAX (#include <stdint.h>) 

Prototype
	void	*ft_calloc(size_t nmemb, size_t size);

[游릭] Happy path
	ft_calloc(2, 1)
		-> 1 arr of 2 bytes initialised to 0

[游릭] No size or members
	ft_calloc(0, 1)
		-> Null OR 1 arr of 0 bytes 
	ft_calloc(SIZE_MAX, 0)
		-> Null OR 1 arr of 0 bytes 
	ft_calloc(0, SIZE_MAX)
		-> Null OR 1 arr of 0 bytes 
			
[游릭] Size too long
	ft_calloc(SIZE_MAX, 1 * sizeof(char))
		-> Null
	ft_calloc(1, SIZE_MAX)
		-> Null
	ft_calloc(SIZE_MAX, SIZE_MAX)
		-> Null
