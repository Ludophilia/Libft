Libft - ft_memset 游릭游댮

Evil tests made with 游땓游땓游땓 passion

Context : The memset() function fills the first n bytes of the memory area
pointed to by s with the constant byte c.

Test material
	- Array of char of 5 elements or space allocated via malloc
	- Original memset (#include <string.h>) for comparison

[游릭] Happy path
	ft_memset(arr, 42, 3)
		-> 3 bytes change in arr

[游릭] Exact size
	ft_memset(arr, '0', 5)
		-> 5 bytes are written

[游릭] Negative int
	ft_memset(arr, -2147, 4)
		-> 4 bytes are written

[游릭] Positive int
	ft_memset(arr, 255, 2)
		-> 2 bytes are written

[游릭] 0
	ft_memset(arr, 0, 1)
		-> 1 byte is written

[游릭] Invalid memory address;
	ft_memset((void *)0, 1, 1)
		-> Segfault / Undefined behavior

[游릭] No size
	ft_memset(arr, '0', 0)
		-> Nothing changes in arr

[游댮] Size too long
	ft_memset(arr, 10, 10)
		-> Write as much as needed (+ Undefined behavior ?)
