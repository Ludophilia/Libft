Libft - ft_strncmp 游릭游댮

Evil tests made with 游땓游땓游땓 passion

Context : The strcmp() function compares the two strings s1 and s2.  The
       locale is not taken into account (for a locale-aware comparison,
       see strcoll(3)).  The comparison is done using unsigned
       characters.
	   
	   The strncmp() function is similar, except it compares only the first 
	   (at most) n bytes of s1 and s2.

Test material
	- read only strings ex : "hello", "world"
	- Original strncmp (#include <string.h>) for comparison

Prototype
	int	ft_strncmp(const char *s1, const char *s2, size_t n);

[游릭] Happy path
	ft_strncmp("hello", "hallo", 1)
		-> 0
	ft_strncmp("hello", "hallo", 3)
		-> 4 (stopped at 'e'/'a')
	ft_strncmp("hello", "hallo", 5)
		-> 4 (stopped at 'e'/'a')

[游릭] Empty strs
	ft_strncmp("", "", 1)
		-> 0
	ft_strncmp("p", "", 1)
		-> 112
	ft_strncmp("", "p", 1)
		-> -112

[游릭] Null address
	ft_strncmp((char *)0, "", 1)
		-> Segfault
	ft_strncmp("", (char *)0, 1)
		-> Segfault
	ft_strncmp((char *)0, (char *)0, 1)
		-> Segfault

[游릭] No size
	ft_strncmp("hello", "hallo", 0)
		-> 0
	ft_strncmp("hello", "yallo", 0)
		-> 0
	ft_strncmp("hello", "", 0)
		-> 0

[游릭] Too much size
	ft_strncmp("hello", "hallo", 10)
		-> 4
	ft_strncmp("hello", "hello", 10)
		-> 0
	ft_strncmp("hello", "", 10)
		-> 104
